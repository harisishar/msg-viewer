# Technology Stack

**Analysis Date:** 2026-04-01

## Languages

**Primary:**
- TypeScript 5.0+ - All source code, build scripts, and type definitions
- HTML - UI markup (`lib/index.html`, component templates)
- CSS - Styling (`lib/resources/styles/styles.css`)

**Secondary:**
- JavaScript - Generated by TypeScript transpilation and bundling

## Runtime

**Environment:**
- Browser (Client-side only) - Application runs entirely in user's browser with no server dependency

**Package Manager:**
- npm - Dependency and lock file management
- Lockfile: `package-lock.json` (present)

## Frameworks

**Core:**
- Bun 1.1+ - JavaScript runtime and build tool, used for bundling and build orchestration (`build.ts`)

**Build/Dev:**
- Bun Build API - Bundler with minification support
- bun-plugin-html 2.0.0 - HTML plugin for Bun build system enabling HTML module support and inlining
- @types/bun latest - Type definitions for Bun runtime

## Key Dependencies

**Critical:**
- @molotochok/msg-viewer 1.0.3 - NPM package containing the core MSG file parsing library
  - Purpose: Provides `parse()` and `parseDir()` functions for parsing Outlook .msg compound files
  - Location: Imported in `lib/scripts/index.ts`

**Development:**
- @types/node 20.12.14 - Node.js type definitions (included as transitive dependency)
- @types/ws 8.5.12 - WebSocket type definitions (included as transitive dependency)
- clean-css 5.3.3 - CSS minification (transitive via bun-plugin-html)
- html-minifier-terser 7.2.0 - HTML minification (transitive via bun-plugin-html)
- linkedom 0.15.3 - DOM manipulation library for HTML processing (transitive via bun-plugin-html)

## Configuration

**TypeScript:**
- Config file: `jsconfig.json`
- Target: ESNext
- Module: ESNext with bundler resolution
- JSX: react-jsx (configured but not actively used)
- Library: ESNext, DOM
- Strict mode: Enabled
- Verbatim module syntax: Enabled for bundler compatibility

**Build:**
- Entry point: `lib/index.html`
- Build config: `build.ts` (Bun build orchestration script)
- Output directory: `build/`
- Minification: Enabled for production
- Plugins: bun-plugin-html with inline option enabled

**Development:**
- No .env files used (client-side only, no server secrets)
- Resource files copied from `lib/resources/` to build output

## Platform Requirements

**Development:**
- Bun runtime (latest stable)
- Node.js 18+ (for npm package management compatibility)

**Production:**
- Deployment: Cloudflare Pages
- Target browsers: All modern browsers supporting ES2020+ (ESNext transpiled)
- Hosting: https://msg-viewer.pages.dev/
