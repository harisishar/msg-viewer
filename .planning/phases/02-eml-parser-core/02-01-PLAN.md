Create `lib/scripts/eml/eml-parser.ts` with the following implementation:

1. **Imports:**
   - `import PostalMime from 'postal-mime'` (default export)
   - `import type { UnifiedMessage, UnifiedMessageContent, UnifiedAttachment, UnifiedRecipient } from '../types/unified-message'`
   - Do NOT import DOMPurify yet — that is added in plan 02-02

2. **Export the main parse function:**
   ```typescript
   export async function parseEml(source: string | ArrayBuffer): Promise<UnifiedMessage>
   ```
   - Create `new PostalMime()` instance
   - Call `const email = await parser.parse(source)`
   - Map to UnifiedMessage using private helper functions (see below)
   - Wrap in try/catch: if postal-mime throws, throw a descriptive error: `"Failed to parse EML file: {original error message}"`

3. **Private helper: `mapContent(email)`** returns `UnifiedMessageContent`:
   - `date`: Parse `email.date` with `new Date(email.date)` — if invalid (isNaN), set to `null`
   - `subject`: `email.subject ?? ''`
   - `senderName`: Extract from `email.from` — postal-mime returns `{ name?: string, address: string }`. Use `email.from?.name ?? ''`
   - `senderEmail`: `email.from?.address ?? ''`
   - `body`: `email.text ?? ''`
   - `bodyHTML`: `email.html ?? ''` (raw HTML — sanitization added in plan 02-02)
   - `bodyRTF`: `undefined` (no RTF in MIME)
   - `headers`: Join `email.headers` array — postal-mime returns `Array<{ key: string, value: string }>`. Map to `"Key: Value"` lines joined with `\n`. If `email.headers` is empty/undefined, use `''`

4. **Private helper: `mapRecipients(email)`** returns `UnifiedRecipient[]`:
   - Map `email.to` (array of `{ name?, address }`) to recipients with `type: 'to'`
   - Map `email.cc` to recipients with `type: 'cc'`
   - Map `email.bcc` to recipients with `type: 'bcc'`
   - Each: `name: addr.name ?? ''`, `email: addr.address ?? ''`
   - Handle undefined arrays (email with no To, CC, or BCC): default to `[]`

5. **Private helper: `mapAttachments(email)`** returns `UnifiedAttachment[]`:
   - Map `email.attachments` array. Each postal-mime attachment has: `{ filename, mimeType, content (ArrayBuffer), disposition, related }`
   - Map to: `fileName: att.filename ?? ''`, `displayName: att.filename ?? ''`, `mimeType: att.mimeType ?? 'application/octet-stream'`, `content: new Uint8Array(att.content)`
   - `embeddedMessage`: `undefined` for now (Phase 4 handles message/rfc822)
   - Handle undefined attachments array: default to `[]`

6. **Error handling:**
   - If `email.html` and `email.text` are both empty/undefined but headers parsed OK: set `body` to `'Body could not be parsed'` and `bodyHTML` to `''`
   - Missing optional fields (CC, Date) produce empty strings / null, never throw

Module structure: single flat file, no subdirectories. Private helpers are not exported. This mirrors msg-parser.ts pattern.
